# 학생맞춤통합지원 웹앱 실행 방법

## 실행
1. `start_webapp.bat`를 더블클릭합니다.
2. 콘솔에 접속 주소가 표시되면 브라우저로 접속합니다.
3. 같은 인트라넷 사용자는 `http://서버PC_IP:포트`로 접속할 수 있습니다.

## 핵심 기능
- 담임교사 제보: 담임교사 성명, 학생 학년/반, 학생명, 발견유형, 담임의견 입력
- 관리자 대시보드:
  - 탭 분리: 신규제보 접수함 / 사례관리
  - 제보 목록 확인
  - 신규 제보 삭제
  - 제보 기반 사례번호 생성
  - 사례번호 코드 변경(SI → 원하는 코드)
  - 필요 시 수동 사례번호 직접 지정
  - 사례 상태(지원 진행/점검/종결) 변경
  - 사례 삭제
  - 관리자 비밀번호 변경(초기값 1234)
- 각 부서 사례번호 접속:
  - 담임교사 의견 확인
  - 부서 지원방향 작성 및 공유

## 데이터 저장
- 서버 파일 `data.json`에 저장됩니다.
- 따라서 같은 서버에 접속한 사용자끼리 사례 데이터가 공유됩니다.

## 문제 해결
- `지원하지 않는 API 경로`가 나오면 서버를 완전히 종료 후 다시 실행하세요.
  - 기존 실행 중인 오래된 서버 버전이 남아 있으면 새 기능(`/api/case/config`, `/api/case/delete`)이 동작하지 않을 수 있습니다.

## 포트 변경
```bat
start_webapp.bat 8080
```


## 방화벽 자동 허용
- `start_webapp.bat` 실행 시, 같은 인트라넷 접속을 위해 **Windows 방화벽 인바운드 규칙(TCP 포트)**을 자동으로 등록하려고 시도합니다.
- UAC(관리자 권한) 팝업이 뜨면 **허용**해야 외부 PC 접속(`http://서버PC_IP:포트`)이 가능합니다.
- 팝업을 취소하면 서버는 실행되지만, 다른 PC에서 접속이 계속 차단될 수 있습니다.

수동으로 열어야 할 때(관리자 CMD):
```bat
netsh advfirewall firewall add rule name="StudentSupportWebApp_8000" dir=in action=allow protocol=TCP localport=8000 profile=private,domain
```
