# 학생맞춤통합지원 협업 웹앱 기획안

## 1. 기획 배경
학생맞춤통합지원은 한 명의 담당자가 모든 업무를 떠안는 방식으로 운영되기보다, 담임교사·상담교사·보건교사·부장교사·관리자·외부기관이 각자의 역할을 나누어 협력하는 구조로 작동해야 한다. 업로드된 2026학년도 학생맞춤통합지원 계획에서도 학교장(위원장), 교감(부위원장), 교무부장, 생활안전부장, 창의인성부장, 진로진학부장, 보건교사, 전문상담교사, 담임교사, 학년부장, 학부모 운영위원, 외부 전문기관 등이 함께 참여하는 팀 체계를 전제로 하고 있다. 또한 정기회와 임시회 운영, 위기 원인 분석, 대상자 선정, 맞춤형 지원 관리, 상담·복지·학습·안전 영역별 역할 분담이 강조된다. fileciteturn1file1 fileciteturn1file3

따라서 웹앱의 목적은 다음과 같다.
- 특정 1인이 독박을 쓰지 않도록 업무를 분산한다.
- 사례번호 중심으로 협업하여 필요한 정보만 공유한다.
- 사례 발견 → 접수 → 회의 → 지원 → 점검 → 종료의 흐름을 기록으로 남긴다.
- 학교 내 인트라넷 환경에서만 접속하도록 하여 보안을 높인다.

## 2. 기본 운영 개념

### 2.1 운영 방식
- 사용자의 컴퓨터 1대를 **학교 내 임시 서버(호스트 PC)** 로 사용한다.
- 같은 인트라넷(교내망) 안에 있는 교직원은 브라우저로 접속한다.
- 담당자는 새 학생 케이스를 생성하고 **사례번호**를 발급한다.
- 다른 교직원은 사례번호를 입력하여 해당 케이스에 접속한다.
- 사례번호만 안다고 해서 모두가 모든 내용을 볼 수는 없고, **권한에 따라 보이는 정보 범위가 달라진다.**

### 2.2 설계 원칙
- **최소 공개 원칙**: 필요한 사람만 필요한 정보만 본다.
- **역할 기반 협업**: 담임, 상담, 보건, 생활지도, 관리자 등 각자 입력한다.
- **기록 자동화**: 누가 언제 무엇을 했는지 자동 저장한다.
- **회의 중심 지원**: 사례회의 결과가 실행 계획으로 이어지게 한다.
- **학교 친화성**: 복잡한 전문용어보다 학교 업무 흐름에 맞춘 UI를 사용한다.

## 3. 기대 효과
1. 학생맞춤통합지원팀의 역할 분담이 실제 업무로 연결된다.
2. 담임교사, 전문상담교사, 보건교사, 부장교사가 동시에 같은 학생을 지원할 수 있다.
3. 회의가 끝난 뒤에도 “누가 무엇을 하기로 했는지” 남는다.
4. 위기학생, 학업중단위기학생, 복지지원학생, 정서지원학생을 한 플랫폼에서 관리할 수 있다.
5. 사후 점검이 쉬워져 행정 보고 및 근거 자료 확보가 수월해진다.

## 5. 핵심 업무 흐름

### 5.1 사례 발굴
계획서에는 담임, 전 교사, Wee클래스, 생활지도, 외부기관 등을 통한 위기 징후 발견이 강조되어 있다. 또한 학생 상담, 학부모 상담, 행동 관찰, 수업 문제, 결석·지각, 가정 상황 파악 등이 주요 발견 경로로 제시되어 있다. fileciteturn1file2 fileciteturn1file4

웹앱에서의 흐름:
1. 교사가 “사례 제보/발견” 버튼 클릭
2. 학생 기본정보 입력
3. 발견 유형 선택
   - 학업
   - 출결
   - 정서·행동
   - 또래관계
   - 학교폭력
   - 자해·자살위험
   - 가정·경제
   - 건강
   - 복합위기
4. 긴급도 선택
   - 일반
   - 주의
   - 긴급
5. 담당자에게 자동 전달

### 5.2 사례 생성
담당자가 검토 후 정식 케이스를 생성한다.
- 사례번호 자동 생성
  - 예: 2026-JD-001
- 사례명 자동 생성
  - 예: 2학년 3반 OOO 학생 통합지원 사례
- 주관 부서 지정
- 참여자 지정
- 최초 회의 필요 여부 선택

### 5.3 사례 참여
교직원은 사례번호를 입력하여 참여한다.
- 사례번호 입력
- 본인 인증 후 접속
- 본인 권한 범위 내 정보만 열람 가능
- 요청받은 항목만 작성 가능

### 5.4 사례회의
계획서상 정기회와 임시회, 위기 원인 분석, 대상자 선정, 맞춤형 지원 관리가 중요하게 제시된다. fileciteturn1file0 fileciteturn1file4

웹앱 기능:
- 사례회의 소집
- 참석자 자동 알림
- 회의자료 자동 모음
- 회의록 작성
- 결정사항 체크리스트화
- 후속 담당자 자동 배정

### 5.5 지원 실행
지원영역별로 실행 내용을 입력한다.
- 상담지원
- 학업지원
- 생활지도
- 복지지원
- 건강지원
- 진로지원
- 외부기관 연계

### 5.6 점검 및 종결
- 점검일 예약

- 추가 회의 필요 여부 판단

## 7. 있으면 좋은 고급 기능

## 7.2 역할별 맞춤 화면
- 담임 화면: 출결, 관찰, 보호자 연락 중심

- 보건교사 화면: 건강·위험 신호 중심

- 사례번호 조회

정도는 되면 실무 활용도가 높다.

## 8. 권한 설계

### 8.1 기본 원칙
민감한 상담 내용은 모든 교사가 볼 수 없어야 한다. 대신 학생 지원에 필요한 요약정보만 공유한다.

### 8.2 권한 예시
#### 총괄관리자
- 전체 현황 조회 가능
- 민감기록 원문은 제한적 열람

#### 사례담당자
- 전체 사례 정보 관리
- 참여자 권한 부여 가능
- 회의록 및 계획 수정 가능

#### 담임교사
- 출결, 행동관찰, 보호자 연락 기록 입력
- 상담 세부 축어록은 비공개
- 지원계획 중 본인 관련 항목만 열람

#### 보건교사
- 건강·위험 관련 항목 열람
- 정서 세부 상담 내용은 제한

#### 부장교사
- 부서 관련 지원 현황 열람
- 민감 세부내용 일부 제한

#### 외부기관 공유용
- 별도 링크나 문서 출력본으로 최소정보만 제공

학생맞춤통합지원은 민감정보를 다루므로 보안 설계가 매우 중요하다.

필수 보안 기능:
- 교내망 외부 접속 차단
- 사용자별 로그인
- 접근기록 저장
- 파일 암호화 저장
- 자동 백업
- 삭제 기록 보존
- 화면 캡처·무단 출력 경고 문구 표시
- 일정 시간 미사용 시 세션 종료
- IP 및 기기 기록 저장

추가 권장:
- 서버 PC 전용 계정 사용
- 관리자 외 USB 저장 제한
- DB 암호화
- 일별 백업 파일 생성

## 12. 실제 학교 적용 시나리오 예시

2. 웹앱에 사례 제보 등록
3. 담당자가 검토 후 사례 생성
4. 사례번호 발급

7. 사례회의 진행
8. 지원계획 수립
9. 2주 뒤 점검

### 시나리오 2. 상담실에서 먼저 접수된 경우
1. 학생이 Wee클래스로 자발적 방문
2. 상담교사가 초기 상담 후 복합위기 판단

5. 필요시 보건, 복지, 외부기관 연계
6. 회의 결과를 바탕으로 통합지원 진행

1. 자해·자살위험 징후 입력 시 자동으로 긴급 표시

3. 긴급회의 소집
4. 조치 내용 실시간 기록

## 14. 제안 결론
이 웹앱의 핵심은 단순한 상담기록 시스템이 아니라, 학생맞춤통합지원 정책이 요구하는 **학교 내 다부서 협업 구조를 실제로 작동하게 만드는 것**이다. 업로드된 계획서에서도 학생맞춤통합지원팀의 다수 구성원 참여, 정기·임시회의 운영, 위기 원인 분석, 맞춤형 지원 관리, 담임·상담·보건·생활·복지·진로 영역별 역할 분담이 제시되어 있으므로, 웹앱 역시 이 구조를 반영해야 한다. fileciteturn1file0 fileciteturn1file1 fileciteturn1file3

즉, 이 시스템은 다음 세 가지를 반드시 달성해야 한다.
1. 사례번호 중심으로 여러 교직원이 함께 일할 수 있어야 한다.
2. 민감정보는 보호하면서도 필요한 협업은 가능해야 한다.
3. 회의와 개입 결과가 실제 실행과 점검으로 이어져야 한다.

이 방향으로 개발하면, 특정 1인이 모든 책임을 지는 구조를 줄이고, 학교 전체가 학생 지원을 공동 책임지는 체계를 만드는 데 도움이 될 것이다.
